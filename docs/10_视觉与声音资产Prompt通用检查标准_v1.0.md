# 《阿修罗》阶段2A｜视觉与声音资产 Prompt 通用检查标准

版本：v1.0（主创确认版）  
建立日期：2026-07-13  
确认日期：2026-07-13  
规则系统：v1.0.1  
模型适配卡：MA-SD2.0-BASE-v1.0  
适用范围：阶段2A测试资产、阶段2B全部视觉与声音资产 Prompt、阶段2C外部生成回流检查  
当前状态：主创已确认；硬性项零容忍，软性项修订复检；待P0实测校准检查表达，不改变检查原则

## 1. 主创确认口径

1. 硬性项零容忍；任一硬性项不合格，Prompt或回流资产即判定不通过。
2. 软性项不得带病通过；修订Prompt并重新生成、复检后，才能进入确认流程。
3. P0资产逐项确认；P1、P2资产按批次确认，但批次确认不降低单个资产的检查标准。
4. 只有经主创确认的资产，才能在可调用视觉资产映射表或可调用声音资产映射表中标记为`已确认可调用`。
5. 失败资产只用于问题分析，不作为后续人物、场景、道具、VFX或声音参考。
6. 检查顺序固定为：生产资产ID与生成目的 → 身份 → 结构与物理 → 材质 → 光影 → 色彩 → 空间与连续性 → 声音 → 禁止项。

## 2. 检查对象与检查时点

本标准同时检查两类对象：

1. **生成前Prompt**：确认输入信息、生产目的和约束没有错误，再交给外部平台生成。
2. **生成后回流资产**：确认实际图像或声音符合Prompt、阶段1资产信息和阶段2A风格锁定口径。

每个资产至少经历以下状态：

```text
待编写 → Prompt预检 → 待生成 → 待回流 → 回流检查 → 待主创确认 → 已确认可调用
                                      ↘ 需修订Prompt → 重新生成与复检
                                      ↘ 暂缓 / 补充拆解
```

不存在“有硬伤但先调用”的中间状态。

## 3. 判定等级

| 判定 | 定义 | 处理 |
|---|---|---|
| 硬性不合格 | 违反项目锁定、稳定ID、人物身份、空间轴线、实景光源、声音策略、禁止改写项或出现不可用结构错误 | 立即停止；不得生成或不得继承回流结果；修订Prompt后重新开始 |
| 软性偏差 | 未改变核心口径，但强度、程度、节奏或质感尚未达到项目目标 | 记录偏差；修订Prompt；重新生成并复检 |
| 待主创确认 | 技术检查通过，但涉及P0身份、风格、关键材质、声音样本或其他主创决策 | 不得标记可调用；提交主创确认 |
| 已确认可调用 | 硬性项全部通过，软性偏差已完成复检，且获得对应级别的主创确认 | 写入映射表，可供后续生产调用 |
| 暂缓 | 信息不足、外部平台能力不足或依赖资产未通过 | 保留记录；不得用Prompt擅自补写 |

## 4. 所有资产 Prompt 的生成前通用检查

### 4.1 字段与版本

- Prompt编号、Prompt版本、生产资产ID、资产类型、资产级别和生成目的完整。
- 规则系统、模型适配卡、风格锁定包和2B分工包版本一致。
- 阶段1来源、必须继承、允许创作补充、禁止改变、输出规格、检查要点和确认状态完整。
- 参考素材必须注明准确名称、版本和用途；不得把失败图或未确认样本写成身份、结构、色彩或声音锚点。
- 一条Prompt只服务一个生产资产ID，或同一资产的一组规定视图／声音样本。
- 信息不足时登记补充拆解项，不以文学性描述或模型猜测补齐事实。

以下情况均为硬性不合格：

- 生产资产ID缺失、错误、被重命名、拆分、合并或替换。
- 混写两个无关资产，导致生成目的不唯一。
- 使用错误规则版本、错误资产卡、错误人物身份或错误场景视图ID。
- 参考素材用途不明，或要求模型同时继承相互冲突的脸、结构、光线、色彩或声音。
- 把创作者建议写成阶段1事实，或擅自改变已确认信息。
- Prompt正文与“必须继承”“禁止改变”相互矛盾。

### 4.2 输出目的

- 人物资产必须区分：剧照验证图、基础身份资产、服装／状态变体、手部或面部局部测试。
- 场景资产必须区分：母图、规定方向视图、空间补全设计卡、光影变体或SV3专项视图。
- 道具、VFX、图案、群演和声音资产必须写明基础资产还是状态／用途变体。
- 静态资产Prompt不预演正式剧情镜头，不混入阶段3镜头运动、剪辑或管理字段。
- 人物、服装和道具基础资产使用中性低反差结构光；场景母图使用已登记的项目实景光源。
- 最终成片的21:9、4K、24fps不强制套用为所有静态资产画布比例；静态资产按其结构验证目的写输出规格，但不得改变最终成片技术口径。

## 5. 视觉资产 Prompt 生成前检查

| 检查维度 | 必须通过 | 硬性不合格示例 | 可修订的软性偏差 |
|---|---|---|---|
| 身份与年龄 | 女主继承同一演员身份；佛国和镜头29为20—22岁；轮回年龄按事件时刻变化 | 换脸、网红脸、错误年龄、用妆发代替真实年龄变化 | 皮肤生活感、疲态、表情幅度需要微调 |
| 面部与手部锚点 | 脸型、眉骨、眼型、唇形、面部比例、手掌比例和主要指节保持稳定 | 改变五官比例、隐藏手部问题、要求宽袖遮手 | 毛孔强度、唇部含水度、手部纹理程度 |
| 结构与接触 | 人体、服装、场景、道具结构明确；重量、受力、持握和接触可成立 | 多指少指、穿模、念珠悬浮、场景门窗冲突 | 褶皱、磨损、接触压力的程度 |
| 材质 | 皮肤、织物、石木、漆面、金属、纸绢和叶片表面反应可区分 | 塑料皮肤、树脂质感、游戏贴图、材料混淆 | 旧化、粗糙度、轻微湿润程度 |
| 光影 | 主光来自已登记的实景光源；人物、道具和场景共享同一光源逻辑 | 无来源补光、金色圣光、美容柔光、HDR提亮、人物独立轮廓光 | 暗部层次、高光滚降、雾或尘的强度 |
| 色彩 | 继承母体中性色、轮回命运主色和青绿落叶权重规则 | 高饱和滤镜、肤色漂移、其他色彩压过青绿落叶 | 命运主色比例、明度和彩度的小幅校准 |
| 空间与视图 | 佛国轴线可追溯；只执行已确认SV0／SV1-lite及后续登记SV3 | 新增SV2或标准五视图、镜像代替左右视图、镜头29新建母图 | 空气透视、可拍摄区和局部陈设密度 |
| VFX与文字 | VFX写成可见物理结果；除片名外文字不可辨 | 佛正脸、怪物化阿修罗、发光眼、能量波、可读诏书或真实印面 | 失焦、层退、落叶下沉速度与程度 |

## 6. 声音资产 Prompt 生成前检查

### 6.1 人物声音

- 女主声音绑定`ASR-VOI-P0-001-女主千年同声纹`，只锁定稳定音色，不固化某场戏的具体情绪表演。
- 明确声音年龄、性别、音域、基础音色、质感、语速、吐字、气声比例和辨识锚点。
- 使用中性测试文本；不得使用会把悲伤、愤怒、哭腔或权力状态固化进基础声纹的测试方式。
- 女主声音由阶段2提供Prompt、用户在外部平台生成并回流确认。
- 佛声和众声优先由正式镜头Prompt控制，不擅自改造成独立声音文件资产。

### 6.2 声音母题与场内声音

- 声音来源、物理发生方式、基础节奏、音域／频率、空间距离和可调整范围明确。
- 念珠、木鱼、梵呗、远磬、风声、呼吸、水滴、心跳、衣料、断弦、印章、酒杯、门轴、风雪和喘息均保持可追溯物理来源。
- 镜头22弦乐必须写为场内同期琴声，不得写成非叙事配乐。
- 镜头23隔门喜乐声和杯沿轻碰写入镜头声音Prompt，不新增声音资产。
- 不使用持续低频铺底代替心理变化。

以下情况均为硬性不合格：

- 错误声音资产ID、错误角色、错误年龄或无法保持稳定的基础音色。
- 基础声音样本固化具体剧情情绪、持续哭腔、甜美少女音、妖艳反派音或播音腔。
- 把佛声、众声或镜头23专项声音擅自改为独立资产。
- 把镜头22琴声写成背景配乐，或加入任何非叙事配乐。
- 无物理来源电子音、持续低频、过强混响掩盖音色、明显削波或不可修复噪声。

可修订的软性偏差包括：语速、字头力度、句尾收束、气声比例、基础距离、混响余量和母题节奏的轻微调整；调整不得改变稳定音色或声音来源。

## 7. 视觉回流资产检查顺序

1. **生产资产ID与目的**：文件、原Prompt和回流用途能一一对应。
2. **身份与年龄**：女主身份、事件年龄、群像差异和佛的非正面存在符合规定。
3. **结构与物理**：人体、手部、服装层次、场景几何、道具比例、受力和接触正确。
4. **材质**：皮肤、织物、石木、漆面、金属、纸绢、叶片和火焰表面真实且可区分。
5. **光影**：实景光源方向、色温、阴影、暗部、高光和空气介质统一。
6. **色彩**：母体中性色、命运主色、肤色和青绿落叶的相对权重正确。
7. **空间与连续性**：场景轴线、视图方位、人物状态、道具状态和REL关系没有冲突。
8. **VFX／文字／图案**：只出现允许的可见结果，文字可读性符合规定。
9. **输出可用性**：清晰度、构图、背景、结构展示和参考价值满足本次生成目的。
10. **禁止项**：逐项核对`docs/11_禁止改写项清单_v1.0.md`。

任何硬性不合格项出现时，停止后续继承；不得用调色、裁切、遮挡或后期滤镜掩盖身份、结构、空间或光源错误。

## 8. 声音回流资产检查顺序

1. 生产资产ID、声音使用目的、原Prompt版本和平台样本对应准确。
2. 声音年龄、音域、基础音色、吐字和辨识锚点稳定。
3. 中性样本没有固化具体剧情情绪，保留后续表演空间。
4. 声音来源和角色／生物体量成立；母题具有清楚的物理来源。
5. 没有持续哭腔、播音腔、甜美少女音、妖艳反派音或无来源电子质感。
6. 没有明显削波、爆音、不可修复底噪、过强混响或压缩泵动。
7. 允许后续调整语速、距离、混响和场次情绪，不需要通过换音色完成剧情变化。
8. 经主创确认后，登记声音使用名、平台引用名和允许用途；未确认样本不得进入阶段3。

## 9. 首批P0测试资产专项检查

### 9.1 `ASR-CHR-P0-001-女主基础身份`

- 先完成真人剧照验证图，再由主创确认脸、年龄、气质和真人感，之后才能建立基础身份资产。
- 同时检查脸型、眉骨、眼型、唇形、面部比例、发际关系、手掌比例和主要指节。
- 佛国视觉年龄为20—22岁；使用真实皮肤、自然不对称和克制表情。
- 女主身份通过前，不扩展轮回服装状态。

### 9.2 `ASR-SCN-P0-002-殿内修行空间`

- 先生成`MASTER`并由主创确认，再生成`AXIS-DOOR`或`HERO-POS`非母图方向视图。
- 母图与方向视图必须继承同一空间结构、殿门方向、女主位置、群像关系、光柱边缘、材质和实景光源。
- 不制作SV2或标准五视图，不以镜像代替真实方向延伸。

### 9.3 `ASR-PRP-P0-001-念珠`

- 单珠视觉尺度为10毫米；Prompt不得写精确珠数。
- 检查珠径、孔洞、线色与柔度、表面磨损、手部接触点和逆拨方向。
- 珠体不得发光、悬浮或呈塑料／宝石首饰质感。

### 9.4 `ASR-PRP-P0-003-青绿落叶`

- 窄椭圆真实叶、冷灰青绿、真实叶脉、轻微湿润、不发光。
- 作为全片唯一明确生命色，建立其他命运主色的相对彩度上限。
- 枝端、下沉、片名影子和掌心接触保持同一叶形与完好状态。

### 9.5 `ASR-VFX-P0-001-主观生理真实目光`

- 主观异常只改变焦点、知觉距离、空间可见宽度和声音距离，不改变世界物理。
- 必须能与同一空间的客观镜头配对；佛没有转头或选择女主。
- 禁止神光、宇宙眼、虹膜图案、发光眼、能量线和暧昧凝视。

## 10. P0、P1、P2确认方式

### P0逐项确认

- 每个Prompt预检结果、每轮回流问题、修订版本和最终可调用结论单独记录。
- 女主、主场景、核心道具、核心VFX和女主声音均需主创明确确认。
- 任一依赖链上游未通过，下游不得开始：人物剧照验证图未通过，不做人物基础资产；场景母图未通过，不做方向视图；女主身份未通过，不扩展轮回服装。
- 3—5个P0测试未完成并通过前，不得进入2B批量生产。

### P1／P2批次确认

- 每个批次提供资产ID清单、检查结果、需修订项、补充拆解项和建议处置。
- 批次内任何硬性不合格资产必须剔出并单独修订，不得随批次整体通过。
- 主创可以对无硬性问题的资产进行整批确认；确认结果仍逐资产写入映射表。
- P1／P2批次确认不得覆盖P0依赖关系或禁止改写项。

## 11. 修订与复检

1. 硬性不合格：拒绝当前Prompt或回流资产，记录原因，修订Prompt并重新生成。
2. 软性偏差：只调整相关强度、程度、节奏或质感字段，不改写稳定ID、身份、结构、光源和声音策略。
3. 同一问题连续两轮未改善时，降低单次处理数量，拆分变量；不得扩大批量。
4. 外部平台能力导致结构性失败时，登记平台限制和替代方案；涉及模型切换必须先提交影响说明并由主创确认。
5. P0测试回流后可以校准检查表述、色域数值、声音距离和材质强度；不得改变本标准的硬性项零容忍原则。
